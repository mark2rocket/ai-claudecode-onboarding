## 1. 챌린지 개요

### 이름 & 슬로건

**메인 이름:** AI 비서 5일 챌린지
**부제:** 월요일에 설치하고, 금요일엔 팀장에게 보고한다
**핵심 슬로건:** "아침마다 메일 50통, 이제는 Claude가 먼저 읽습니다."

**가치 제안 문구 (상황별):**
- "5일 후, 당신의 아침 1시간이 달라집니다."
- "AI 도입하라는데 어디서 시작할지 모르겠다면 — 여기서 시작하세요."
- "ChatGPT는 질문에 답하지만, Claude Code는 내 파일을 직접 읽고 메일을 분류하고 보고서를 만드는 로컬 AI 비서입니다."

### 타겟 참여자

**1차 타겟 (코어):**
- 30대 초중반 직장인 (이 대리 ~ 김 과장급)
- 스타트업·중소기업·에이전시 근무자
- ChatGPT를 써봤지만 "제대로 활용 못한다"고 느끼는 사람
- 반복 업무(보고서, 메일, 리서치)에 지쳐있는 사람

**2차 타겟 (확장):**
- 1인 크리에이터·프리랜서 (콘텐츠 자동화 니즈)
- 40대 사무직·경영지원팀 (메일 브리핑, 회의록 자동화)
- AI 도입을 검토 중인 팀장급

### 기간 & 형식

- **기간:** 5일 (월~금)
- **일일 소요 시간:** 25~35분 (Day 3는 45~60분 솔직하게 안내)
- **참여 방법:** 카카오톡 오픈채팅방 + 노션 아카이브
- **인증 방법:** 매일 실습 결과 스크린샷 채팅방 공유
- **권장 시작 시간:** 오전 9시~10시 또는 점심시간

### Claude Code 비용 안내 (선결 조건 — 랜딩 최상단 필수)

> **이 챌린지를 시작하기 전 반드시 확인하세요.**

### Day 1: "AI 비서 입사 첫날 — 온보딩 문서 만들기"

**배울 것:** Claude Code를 설치하고 CLAUDE.md를 작성하여, Claude가 내 직무 맥락에 맞는 언어로 응답하도록 설정할 수 있다.

**핵심 개념:**

- **"AI는 새 직원이다":** ChatGPT는 매번 처음 만나는 프리랜서입니다. Claude Code에 CLAUDE.md를 작성하면, 내 업무 방식을 기억하는 전담 비서가 됩니다. CLAUDE.md는 비서에게 주는 업무 매뉴얼입니다.

- **보안에 대한 정확한 이해:** Claude Code는 내 컴퓨터에서 실행됩니다. 파일을 읽고 쓰는 작업은 로컬에서 처리됩니다. 단, Claude에게 분석을 요청할 때 해당 텍스트는 Anthropic API를 거칩니다. 개인정보나 기밀이 포함된 내용은 익명화 후 입력하세요.

**에러 대응 공식 (모든 Day 공통):**
> **에러 메시지가 나왔나요? 그 메시지를 그대로 복사해서 Claude에게 붙여넣고 "이 에러 해결해줘"라고 하세요.** 이 한 줄이 비개발자의 자생력을 결정합니다.

**만드는 파일들:**

폴더 구조 생성 프롬프트 (Claude에게 입력):
```
~/ai-assistant 폴더를 만들고, 그 안에 inbox, output, research, skills 폴더를 만들어줘.
각 폴더 안에 .gitkeep 빈 파일도 만들어줘.
```

**CLAUDE.md 전체 내용** (`~/ai-assistant/CLAUDE.md`로 저장. 대괄호 부분을 본인 정보로 교체):

```markdown
# 나의 AI 비서 설정

## 역할
당신은 15년 경력의 꼼꼼한 시니어 비서입니다.
보고서는 반드시 '결론 → 근거 → 다음 액션' 순서로 작성합니다.
한국어로 대화하고, 격식체(~입니다/~합니다)를 유지합니다.

## 내 정보
- 이름: [이름 입력]
- 직책: [직책 입력] (예: 마케팅팀 이 대리, PM, 경영지원 대리)
- 회사 유형: [회사 설명] (예: 직원 50명 뷰티 이커머스 스타트업)
- 주요 업무:
  - [업무1] (예: 퍼포먼스 마케팅 운영)
  - [업무2] (예: SNS 콘텐츠 기획 및 업로드)
  - [업무3] (예: 월간 성과 보고서 작성)

## 협업 툴
- 슬랙: [주 사용 채널] (예: #marketing, #general)
- 노션: [주 사용 DB] (예: 콘텐츠 캘린더, OKR 트래커)
- 이메일: [Gmail / Outlook 중 선택]
- 기타: [Google Sheets, Linear 등]

## 출력 스타일
- 길이: 핵심만, 불필요한 감탄사 없이
- 형식: 불릿포인트 우선, 숫자는 원화(₩) 표기
- 보고서: 3줄 요약 먼저, 그다음 상세 내용
- 이메일 초안: 수신자 직급에 맞는 존댓말 수준 자동 조정

## 절대 하지 않는 것
- 확인되지 않은 정보를 사실처럼 말하지 않기
- "~인 것 같아요" 등 추측성 표현 금지 (모를 때는 "확인이 필요합니다")
- 개인정보(주민번호, 계좌번호, 비밀번호)를 외부로 전송하지 않기
- 파일 삭제 작업은 반드시 확인 후 실행

## 현재 진행 중인 프로젝트
- [프로젝트명]: [한 줄 설명]

## 나의 글쓰기 스타일
[내가 쓴 실제 보고서 문장 하나를 여기에 붙여넣기]
(예: "전월 대비 CTR 0.3%p 상승하였으며, 주요 원인은 소재 교체로 판단됩니다.")
```

**실습 태스크 (25분):**

```
[Step 1 — 5분] WOW 먼저 보기
inbox 폴더에 업무 관련 파일 3개를 넣고 입력:
"inbox 폴더에 있는 파일들을 열어서, 각 파일의 핵심 내용을 3줄로 요약해줘.
 공통적으로 개선이 필요한 부분이 있다면 알려줘.
 결과는 output/day1-summary.md 파일로 저장해줘."
→ Claude가 파일을 직접 읽고 요약하는 것을 목격

[Step 2 — 5분] 설치 확인 및 터미널 첫 실행
- 터미널(또는 Warp) 열기 → claude 입력
- "안녕, 나는 마케터야. 오늘 할 일 목록 3개 만들어줘" 첫 대화

[Step 3 — 15분] CLAUDE.md 작성
위 템플릿을 내 정보로 채우기
저장 위치: ~/ai-assistant/CLAUDE.md

[Step 4 — 검증]
"내 CLAUDE.md 내용을 바탕으로, 내가 이번 주 집중해야 할 것이 뭔지 알려줘."
→ 자기 맥락으로 답변이 나오면 성공
```

**성공 체크포인트:**

- "이게 진짜 나를 알고 있네"라는 감각이 한 번 이상 있었다

**WOW 모멘트:** CLAUDE.md 저장 직후, "내 업무에서 자동화할 수 있는 것 3가지를 추천해줘"를 입력. Claude가 CLAUDE.md에 쓴 직무 맥락을 기반으로 구체적인 추천을 내놓는 순간. "이게 진짜 나를 알고 있네."

**에러 대응:** → 에러 메시지가 나오면 그대로 Claude에게 붙여넣고 "이 에러 해결해줘"라고 하세요.

### Day 2: "리서치 인턴 투입 — 60초 경쟁사 분석"

### Day 3: "AI가 내 메일함을 열다 — /mail-briefing"

**배울 것:** Gmail MCP를 연결하고 /mail-briefing 스킬을 실행하여, 매일 아침 1시간 걸리던 메일 정리를 5분 안에 완료할 수 있다.

**Day 3 특별 주의사항:**
> "오늘 Day 3가 이번 챌린지에서 가장 어렵습니다. 에러가 한 번은 납니다. 그게 정상입니다. 이 고비만 넘으면 Day 4·5는 훨씬 쉬워집니다."
> **솔직한 시간 안내: 오늘은 45~60분이 소요됩니다.** (텍스트 가이드만 있을 경우 더 걸릴 수 있습니다)

**핵심 개념:**
- **"MCP = AI의 손":** 지금까지 Claude는 눈만 있었어요. 내가 텍스트를 복붙해줘야 읽을 수 있었죠. MCP를 연결하면 Claude에게 손이 생깁니다. 직접 Gmail을 열고, 읽고, 분류해요.

**Gmail MCP 설치 단계별 가이드:**

**Step 1: Google Cloud Console에서 OAuth 앱 만들기**
1. https://console.cloud.google.com 접속 (Google 계정 로그인)
2. 상단 프로젝트 선택 → "새 프로젝트" → 이름: `claude-assistant` → 만들기
3. 좌측 메뉴 → "API 및 서비스" → "라이브러리"
4. 검색창에 `Gmail API` → 클릭 → "사용 설정"
5. 좌측 메뉴 → "사용자 인증 정보" → "+ 사용자 인증 정보 만들기" → "OAuth 클라이언트 ID"
6. 처음이면 "동의 화면 구성" 먼저 진행:
   - User Type: 외부 → 만들기
   - 앱 이름: `Claude Assistant`, 이메일 입력 → 저장 후 계속
7. 애플리케이션 유형: "데스크톱 앱" → 이름: `Claude Desktop` → 만들기
8. JSON 다운로드 → 파일 이름을 `google-credentials.json`으로 변경
9. 터미널에서 실행:
   ```bash
   # 이 명령어는 다운로드된 파일을 ai-assistant 폴더로 이동하는 명령어입니다
   mv ~/Downloads/google-credentials.json ~/ai-assistant/
   ```

**Step 2: Gmail MCP 패키지 설치**

> **런칭 전 운영자 필수 확인:** 아래 패키지가 실제 npm에서 설치 가능한지, 최신 Claude Code에서 정상 작동하는지 반드시 실제 테스트로 검증하세요. 공식 MCP 서버 목록: https://github.com/modelcontextprotocol/servers

```bash
# 이 명령어는 Gmail MCP 서버 패키지를 전역으로 설치하는 명령어입니다
npm install -g @modelcontextprotocol/server-gmail

# 설치 확인 (버전 번호가 나오면 성공)
npx @modelcontextprotocol/server-gmail --version
```

**Step 3: Claude Code MCP 설정 파일 수정**

먼저 사용자 이름 확인:
```bash
# 이 명령어는 현재 사용자의 홈 폴더 경로를 보여주는 명령어입니다
echo $HOME
# 출력 예시: /Users/jieun  ← jieun이 사용자 이름입니다
```

**Claude Code에게 자연어로 파일 수정 요청 (권장 방법):**
```
~/.claude.json 파일에 아래 내용을 추가해줘.
파일이 없으면 새로 만들어줘.

{
  "mcpServers": {
    "gmail": {
      "command": "npx",
      "args": [
        "@modelcontextprotocol/server-gmail",
        "--credentials-file",
        "/Users/[사용자이름]/ai-assistant/google-credentials.json"
      ]
    }
  }
}

[사용자이름] 부분은 아까 확인한 내 사용자 이름으로 바꿔줘.
```

**직접 편집이 필요한 경우 (대안):**
```bash
# 이 명령어는 nano 텍스트 편집기로 파일을 여는 명령어입니다
nano ~/.claude.json
# 위 내용 붙여넣기 → Ctrl+X → Y → Enter
```

**Step 4: 인증 실행 및 확인**

claude
# 자동으로 브라우저 인증 창이 열림 → Google 계정 로그인 → 권한 허용
```

# 메일 브리핑 스킬

## 실행 순서

1. Gmail MCP를 통해 오늘 받은 메일(최근 24시간)을 모두 읽는다.
2. 각 메일을 4가지 분류 기준으로 나눈다:
   - 긴급·액션 필요: 상사/임원 발신, 마감 언급, 계약/법무 관련
   - 협력사·파트너: 외부 파트너, 검토 요청, 회신 필요
   - 정기 보고·참고: 자동 발송 리포트, 사내 공지, FYI 성격
   - 필터링 권장: 광고, 뉴스레터 구독, 수신 동의 없는 외부 발신

3. 아래 형식으로 출력한다:

---
메일 브리핑 ([오늘 날짜] 기준, 총 [N]통)

긴급·액션 필요 ([N]건)
- [발신자] "[제목 요약]" → 마감: [날짜 또는 "오늘"] / 액션: [해야 할 일 한 줄]

협력사·파트너 ([N]건)
- [발신자] "[제목 요약]" → [요청 내용 한 줄]

정기 보고·참고 ([N]건)
- [제목들 나열, 쉼표 구분]

필터링 권장 ([N]건)
- [간략 설명]

오늘 반드시 처리할 메일: [긴급 건 목록]
예상 소요 시간: 약 [N]분
---

4. 출력 후: "특정 메일에 대해 답장 초안이 필요하시면 번호를 말씀해 주세요." 안내한다.
```

**WOW 모멘트:** `/mail-briefing` 실행 후 긴급 항목에 실제로 오늘 급한 메일이 잡혀 있는 순간. "이게 진짜로 내 메일을 읽은 거야?"

**에러 대응:** → 에러 메시지가 나오면 그대로 Claude에게 붙여넣고 "이 에러 해결해줘"라고 하세요.

### Day 4: "AI 세 개가 동시에 일하다 — daily-briefing"

**배울 것:** Claude가 세 작업을 병렬로 처리하여 Slack·Notion·Calendar를 동시에 분석하고, 45초 안에 통합 데일리 브리핑을 생성할 수 있다.

**핵심 개념:**

- **"병렬 처리 = 팀원에게 업무 위임":** 내가 매니저라면, 슬랙 정리는 막내 김 사원에게, 노션 현황은 이 사원에게, 일정은 박 사원에게 각각 맡기겠죠. Claude가 세 작업을 동시에 처리하면 45초면 됩니다. 내가 순서대로 하면 90분이에요.

- **정확한 이해:** Claude가 세 작업을 동시에(병렬로) 처리합니다. 슬랙·노션·캘린더를 각각 독립된 컨텍스트에서 동시에 읽어오는 방식입니다. (독립된 AI 3개가 아니라, Claude가 효율적으로 병렬 처리하는 것입니다.)

**추가 MCP 설치:**

Slack Bot Token 발급:
1. https://api.slack.com/apps → "Create New App" → "From scratch"
2. App Name: `Claude Assistant`, 워크스페이스 선택 → 만들기
3. 좌측 "OAuth & Permissions" → "Bot Token Scopes" → 아래 4개 추가:
   - `channels:history` / `channels:read` / `chat:write` / `users:read`
4. 상단 "Install to Workspace" → 허용
5. "Bot User OAuth Token" (`xoxb-`로 시작) 복사

Notion Integration 발급:
1. https://www.notion.so/my-integrations → "+ New integration"
2. Name: `Claude Assistant`, 워크스페이스 선택 → Submit
3. "Internal Integration Token" (`secret_`로 시작) 복사
4. **중요 (이 단계를 빠뜨리면 아무것도 읽지 못합니다):**
   Notion에서 연결할 페이지 열기 → 우상단 "..." → "Connections" → "Claude Assistant" 추가

MCP 패키지 설치:

# 데일리 브리핑 스킬

## 개요
3개 작업을 병렬로(동시에) 처리한다.
각 작업은 독립된 컨텍스트에서 실행되므로 서로의 정보에 방해받지 않는다.

## 실행 방법

아래 3개 Task를 동시에 실행한다.

**Task 1 — Slack 요약:**
Slack MCP를 사용한다.
지난 16시간 동안 활동이 있었던 주요 채널을 확인한다.
읽지 않은 메시지와 나에게 직접 언급된 메시지를 가져온다.
채널별로 핵심 내용을 1~2줄로 요약하고, 멘션된 항목은 별도 표시한다.
형식: [슬랙 요약] - #채널명: 핵심 내용 한 줄

**Task 2 — Notion 현황:**
Notion MCP를 사용한다.
오늘 날짜 기준으로 마감이 임박한 항목(오늘 또는 내일)을 찾는다.
형식: [노션 현황] - 오늘 마감: / 검토 필요: / 이번 주 예정:

**Task 3 — 캘린더:**
Google Calendar MCP를 사용한다.
오늘의 일정을 시간순으로 가져온다.
형식: [오늘 일정] - HH:MM [일정 제목] ([소요 시간])

## 출력 형식

---
데일리 브리핑 ([오늘 날짜 요일 포함])

[슬랙 요약]
(Task 1 결과)

[노션 업무 현황]
(Task 2 결과)

[오늘 일정]
(Task 3 결과)

[오늘의 액션 Top 3]
1.
2.
3.
---
```

**성공 체크포인트:**
- `/daily-briefing` 실행 시 3개 소스 정보가 통합 출력된다
- 오늘 집중 TOP 3가 본인의 실제 우선순위와 일치한다 (80% 이상)
- 실행 시간을 측정했을 때 수동 대비 현저히 빠르다

**WOW 모멘트:** 타이머를 켜고 `/daily-briefing` 실행. 45초 후 Slack·Notion·Calendar가 합쳐진 브리핑이 나오는 순간. "내가 커피 내리는 동안 Claude가 세 작업을 동시에 처리했다."

**에러 대응:** → 에러 메시지가 나오면 그대로 Claude에게 붙여넣고 "이 에러 해결해줘"라고 하세요.
- MCP가 안 보일 때 → JSON 문법 오류 확인 (jsonlint.com)
- Notion에서 아무것도 안 읽힐 때 → Notion 페이지에 Claude Assistant Connections 추가 확인
- Slack에서 채널이 없다고 나올 때 → Bot을 해당 채널에 초대했는지 확인

**Day 4 완료 시 배포:** `/start` 뼈대 파일을 지금 배포합니다. Day 5는 이 파일을 채우기만 하면 됩니다.

**예상 소요 시간:** 30분 (MCP 추가 설치 포함 시 45분)

### Day 5: "발표 전날 완성 — /start로 하루를 여는 AI 비서"

**배울 것:** /start 마스터 스킬을 만들어, 매일 아침 한 번의 명령으로 mail-briefing·daily-briefing·오늘의 작전계획이 자동 실행되는 개인 워크플로우를 완성한다.

**핵심 개념:**
- **"Day 5에 새 개념 없음"** (의도적 설계): Day 5는 Day 1~4에서 배운 것들을 연결하는 날입니다. 새로운 이론 없이, 본인이 만든 스킬들을 조합하는 것만으로 완성됩니다. "이미 다 배웠어요. 오늘은 연결만 하면 됩니다."

# 출근 루틴 스킬 (/start)

## 실행 순서

오늘 날짜와 현재 시각을 확인한다.

**Step 1 — 메일 브리핑**
/mail-briefing 스킬을 실행한다.

**Step 2 — 데일리 브리핑**
/daily-briefing 스킬을 실행한다.

**Step 3 — 오늘의 작전 계획 생성**
Step 1과 Step 2 결과를 종합해서 아래 형식으로 출력한다:

---
오늘의 작전 계획 ([날짜] [요일])

지금 당장 해야 할 것
1. [메일/슬랙에서 파악한 긴급 액션]
2. [노션/캘린더에서 파악한 마감 임박 항목]
3. [오늘 미팅 전 준비사항]

오후에 처리할 것
- [중요하지만 급하지 않은 항목들]

오늘 넘겨도 되는 것
- [D+1 이상 여유 있는 항목들]

오늘의 주요 일정
[캘린더 일정 요약]
---

**Step 4 — 결과 저장**
작전 계획을 ~/ai-assistant/output/start-[YYYYMMDD].md 파일로 저장한다.

[선택] 같은 내용을 Slack #daily-log 채널에 포스팅해줘. (빠른 참여자만)
```

**실습 태스크 (35분):**
```
[Morning Retrieval — 10분] ← 오늘은 5일 전체 복습
"나만의 AI 비서 시스템 지도 그리기"
A4 종이에 5일 동안 만든 것들의 연결 구조 손으로 그리기:
CLAUDE.md → /research → /mail-briefing → /daily-briefing → /start

[Step 1 — 5분] 최종 완성본 데모 영상 시청
"/start를 실행하면 이런 일이 일어납니다"
"오늘 이것을 직접 만들 겁니다"

[Step 2 — 15분] /start 스킬 완성
Day 4 종료 시 배포된 뼈대 파일을 채우기
위 SKILL.md 내용으로 저장

[Step 3 — 10분] 실행 + 발표 준비
/start 최종 실행
→ 화면 캡처
→ "오늘 오후 4시 팀장 발표용 한 줄 설명 작성":
  "저는 이번 주에 [이것]을 만들었고, [이렇게] 시간을 절감할 수 있습니다."
```

**성공 체크포인트:**
- `/start` 한 번으로 메일 브리핑 + 데일리 브리핑 + 작전계획이 순서대로 출력된다
- "오늘 절대 놓치면 안 되는 것 1가지"가 본인 판단과 일치한다
- 팀장 발표용 한 줄 설명을 완성했다

**WOW 모멘트:** `/start` 실행. 메일 브리핑이 나오고, 데일리 브리핑이 나오고, "오늘의 작전계획"이 나오는 순간. "나는 아무것도 안 했는데 하루 계획이 다 나왔어." 그리고 이것이 첫날부터 내가 만든 것들이 연결된 결과라는 것을 인식하는 순간.

**에러 대응:** → 에러 메시지가 나오면 그대로 Claude에게 붙여넣고 "이 에러 해결해줘"라고 하세요.
- Day 3~4 MCP가 일부 실패한 경우 → Gmail만 있어도 /start 기본 버전 완성 가능. 작전계획에 메일 브리핑만 반영됩니다.
- Scope creep 방지: "/start는 정확히 3단계만 합니다. 더 추가하지 마세요."

**선택형 확장 (빠른 참여자만):**
1. https://api.slack.com/apps → 앱 선택 → "Incoming Webhooks" 활성화
2. "Add New Webhook to Workspace" → 채널 선택 → Webhook URL 복사
3. /start SKILL.md Step 4 뒤에 추가:
```markdown
[선택] "슬랙에 공유해줘"라고 하면 오늘의 작전계획 3줄 요약을 Slack에 포스팅한다.
```

**예상 소요 시간:** 35분

## 7. 기술 가이드

### MCP 패키지 검증 (런칭 전 운영자 필수 완료)

> Anthropic CTO 지적 사항: `@modelcontextprotocol/server-gmail`, `@modelcontextprotocol/server-slack`, `@modelcontextprotocol/server-notion`, `@modelcontextprotocol/server-google-calendar` 패키지들이 실제로 npm에서 설치 가능한지, 최신 Claude Code 버전에서 정상 작동하는지를 반드시 실제 테스트로 검증하세요.
>
> 공식 MCP 서버 목록: https://github.com/modelcontextprotocol/servers
>
> 커뮤니티 패키지라면 "Anthropic 공식 지원이 아닙니다"라고 명시하고, 동작 보장이 어렵다면 수동 대안(로컬 텍스트 파일)을 기본 경로로 올려야 합니다.

### 보안 Q&A (참여자 자주 묻는 질문)

**Q: "회사 내부 데이터를 Claude Code에 입력하면 외부로 전송되나요?"**

A: 파일을 읽고 쓰는 작업은 내 컴퓨터 안에서 처리됩니다. 단, Claude에게 "요약해줘", "분석해줘" 등 질문을 입력할 때, 해당 텍스트는 Anthropic API를 통해 처리됩니다. 따라서:
- 파일 경로, 폴더 구조: 로컬에서만 처리
- 파일 내용 중 Claude에게 보낸 부분: API로 전송됨
- API 전송 데이터는 Anthropic Privacy Policy에 따라 처리되며, 훈련 데이터로 사용하지 않도록 설정 가능

실무 권고:
- 민감도가 높은 개인정보(주민등록번호, 계좌번호 등)는 익명화 후 입력
- CLAUDE.md에 `내부 기밀 데이터를 외부로 전송하지 않기` 규칙 명시
- 중견기업 이상 또는 금융·의료·공공기관 소속이라면 반드시 IT 보안팀 확인 후 사용

**Q: "Gmail MCP를 연결하면 모든 메일이 자동으로 저장되나요?"**

A: 아닙니다. `/mail-briefing`을 실행할 때만 Gmail API를 호출해서 메일을 읽습니다. 자동 수집·저장 기능은 없습니다. 권한 확인 및 취소: https://myaccount.google.com/permissions

**Q: "Slack Bot이 연결되면 모든 채널을 볼 수 있나요?"**

A: Bot이 초대된 채널만 볼 수 있습니다. 민감한 채널(인사, 급여 등)에 Bot을 초대하지 않으면 됩니다. 최소 권한(`channels:history`, `channels:read`)만 설정하는 것을 권장합니다.

### Windows 사용자 대응 방침

**1기 정책:** Mac 우선 지원. Windows 사용자 참여 가능하나, Day 3(Gmail MCP) 구간에서 폴백 트랙으로 전환.

**폴백 트랙 안내:**
- Day 3: Gmail 연결 대신 샘플 메일 텍스트 파일로 실습

- 핵심 학습 목표(메일 분류 로직, 브리핑 구조)는 동일하게 달성 가능

**2기 대응:** Windows 전용 가이드 제작 후 정식 지원 예정.

## 10. 검수단 합의 요약

### 통과 근거

4인 검수단이 공통으로 인정한 이 챌린지의 강점:

1. **누적 빌드 구조:** Day 1에서 만든 CLAUDE.md가 Day 5의 /start 안에서 살아 움직이는 구조. "버리는 실습"이 없고 "쌓이는 시스템"을 만든다는 감각이 완주율의 핵심 엔진이다. (Lenny, Anthropic CTO 모두 최우선 유지 항목으로 지목)

2. **WOW First 원칙:** 이론보다 충격이 먼저. 매일 첫 5분에 데모를 배치하여 학습 동기를 만든다. 데모 영상이 없는 Day는 없어야 한다.

3. **Day 3 집중 지원 설계:** 가장 어려운 날에 가장 많은 리소스(사전 설치 숙제 + 영상 가이드 + 실시간 Q&A + 폴백 시나리오)를 투입한다. Day 3 오후 7시 긴급 Q&A는 단일 장치로 완주율에 가장 큰 영향을 미친다.

4. **Day 5에 새 개념 없음:** 학습과학 원칙에 맞는 올바른 설계. 4인 전원 동의.

5. **"금요일 오후 4시 발표" 내부 데드라인:** 5일 내내 긴장감을 유지시키는 스토리 아크의 핵심 엔진.
