## EXPLAIN

### Subagent란?

특정 작업만 전담하는 **분업 도우미 Claude**입니다.

메인 대화를 방해하지 않고, 별도의 독립 공간에서 작업한 뒤 결과만 가져다 줍니다.

> Subagent [쉬운 말: 분업 도우미] — 메인 Claude가 특정 작업을 독립된 공간에서 처리하도록 보내는 보조 Claude 인스턴스
> 프로세스 격리 [쉬운 말: 독립 작업 공간] — Subagent는 메인 대화의 내용을 모른 채 새로운 빈 공간에서 시작합니다

---

### 비유로 이해하기

> 팀장(메인 Claude)이 팀원(Subagent)에게 "이 보고서 분석해서 요약해줘"라고 말하는 것.
> 팀원은 별도 공간에서 작업하고, 3줄 요약만 가져다 줍니다.

```
메인 Claude (대화 맥락 유지)
  [A, B, C, D, E 대화 내용...]
       │
       │ "이 100페이지 PDF 분석해줘"
       ▼
  Subagent (빈 공간에서 시작)
  [빈 상태에서 PDF만 분석]
       │
       │ "3줄 요약 결과"
       ▼
  메인 Claude (결과 수신, 대화 이어감)
```

---

### 왜 필요한가?

| 상황 | Subagent 없이 | Subagent 있을 때 |
|------|--------------|-----------------|
| 긴 문서 분석 | 대화 기억 용량 소진 | 별도 공간에서 처리, 요약만 받음 |
| 여러 작업 동시 진행 | 순서대로 하나씩 | 병렬로 동시에 처리 |
| 전문 작업 위임 | 메인 대화 복잡해짐 | 깔끔하게 분리 |

---

### 컨텐츠 크리에이터 활용 예시

```
지난 1년치 유튜브 댓글 100개를 분석해서 시청자들이 가장 원하는 콘텐츠 주제 5개를 뽑아줘
```

→ Subagent가 독립적으로 댓글 분석 후 결과만 전달
