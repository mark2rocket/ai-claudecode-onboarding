### 텍스트 vs HTML

```
Block 3 결과물  →  텍스트 스크립트 (읽는 것)
Block 4 결과물  →  브라우저에서 열리는 카드 (보는 것)
```

---

### 구성을 명시하는 이유

Block 3에서 Claude가 구성을 잘 추측했지만,
**명시하면 매번 동일한 구조가 보장된다.**

```
명시 없음  →  Claude가 추측 (이번엔 맞을 수도, 다음엔 다를 수도)
명시 있음  →  항상 동일한 리듬 (제목 → 내용 → 요약)
```

CTA는 마지막 카드에만. 중간에 넣으면 독자의 주의가 분산된다.

## EXECUTE

**프롬프트 실행 (Block 3 스크립트 → HTML 카드):**

```
30대 직장인을 위한 AI 트렌드 카드뉴스 5장을 HTML로 만들어줘.
1장: 표지, 2장: AI가 바꾸는 직장생활, 3장: AI 도구 3가지,
4장: 생존 전략, 5장: CTA.
1~4장은 제목+핵심 내용 1~2줄+핵심 요약 한 줄로,
5장은 제목+핵심 내용+행동 유도 문구로 구성해줘.
cardnews.html로 저장하고 브라우저로 바로 열어줘.
```

> Claude Code가 파일을 저장하고 브라우저를 자동으로 띄워준다.

**PNG로 저장하기 (SNS 업로드용):**

브라우저에서 확인했으면, 실제 인스타그램에 올리려면 PNG 이미지가 필요하다.

```
cardnews.html의 각 카드를 1080×1080px PNG 이미지로 저장해줘.
파일명은 card-01.png, card-02.png ... 형식으로.
```

> Claude Code가 Puppeteer로 각 카드를 자동 캡처해서 PNG 파일로 저장한다.

```
HTML  →  브라우저 확인 (지금까지)
PNG   →  SNS 업로드 가능 (실제 배포)
```
