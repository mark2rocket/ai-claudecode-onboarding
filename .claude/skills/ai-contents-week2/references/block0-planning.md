**Claude는 빠진 정보를 추측으로 채운다.**
기획은 그 추측을 내가 통제하는 것이다.

---

### 웹앱도, 서비스도, 카드뉴스도 같은 원칙

```
기획서 없이 개발 시작  →  리빌드 반복
스펙 없이 카드뉴스 요청  →  수정 반복

공통점: 빠진 정보를 Claude(또는 개발자)가 추측으로 채운다
해결책: 추측이 필요없도록 스펙을 먼저 정의한다
```

## EXECUTE

**먼저 스펙 없이 날것으로 요청해보자:**

```
카드뉴스 이미지 만들어줘
```

실행 후 아래 체크리스트로 평가한다:

```
□ 대상이 내가 원하는 사람인가?
□ 장수가 내 의도와 맞나?
□ 각 카드 구성이 통일됐나?
□ 디자인이 내 스타일인가?
```

> 4개 중 몇 개나 내 의도대로 나왔는지 확인해보자.
> 이 체크리스트가 Block 1~5의 학습 지도다.
