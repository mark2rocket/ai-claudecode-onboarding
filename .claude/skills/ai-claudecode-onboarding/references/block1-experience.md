# Block 1: 7일 후 미리 체험

> **Phase A 시작 시 반드시 아래 형태로 출력한다:**
> ```
> 📖 공식 문서: https://code.claude.com/docs/ko/quickstart
> ```

> 7일 후 모습을 먼저 체험한다. 외우지 않는다. 느끼기만 한다.

## EXPLAIN

먼저 아래 Before/After 테이블을 보여준다:

| Before (지금) | After (7일 후) |
|---------------|----------------|
| SNS·블로그·뉴스레터 플랫폼 하나씩 열어서 30분 | 한 문장으로 주간 컨텐츠 현황 자동 정리 |
| "이번 주 콘텐츠 계획 세워줘" → 뭘 만들지 방향 없음 | 모호한 요청 → Claude가 질문으로 구체화 |
| 기능 외우려고 문서 뒤지기 | 모르면 Claude에게 물어보기 |

3가지 데모를 안내한다:

---

### 데모 1: Skill 실행 — `/content-weekly-sync`

> 지금 당장 실행 가능합니다. 한번 해보세요.

터미널에서 아래를 입력합니다:

```
/content-weekly-sync
```

Claude가 이번 주 작업 내용을 정리하고, 컨텐츠 현황을 한 문서로 만들어주는 과정을 보여줍니다.

**핵심 포인트:** 한 마디면 끝. 7일 후에는 여러분이 이런 자동화 레시피를 직접 만들 수 있습니다.

---

### 데모 2: 모호한 요청 → AskUserQuestion [쉬운 말: 명확화 질문]으로 구체화

먼저 이렇게 **모호하게** 요청해보세요:

```
이번 달 SNS 콘텐츠 전략 만들어줘
```

Claude가 혼자 추측해서 결과를 냅니다.

이번엔 다르게 요청해보세요:

```
이번 달 SNS 콘텐츠 전략 만들어줘. 모호한 부분은 AskUserQuestion으로 질문해서 명확하게 만들어
```

Claude가 "어떤 플랫폼인가요?", "타겟 독자는 누구인가요?", "게시 빈도는?" 같이 **선택지를 제시하며 질문**하는 걸 체험합니다.

**핵심 포인트:** AskUserQuestion을 활용하면 Claude가 **추측 대신 질문**합니다. 결과의 정확도가 완전히 달라집니다.

---

### 데모 3: Claude에게 물어보기

```
Claude Code에서 MCP가 뭐야?
```

> MCP [쉬운 말: 외부 앱 연결 플러그]는 Block 3-3에서 배웁니다. 지금은 모르는 것을 Claude한테 물어보면 된다는 걸 체험하는 것으로 충분합니다.

**핵심 메시지:** 기능을 다 외울 필요 없습니다. 모르면 Claude에게 물어보면 됩니다.

---

## EXECUTE

1. **데모 2** 직접 실행: 모호하게 요청 → 그다음 AskUserQuestion 버전으로 재요청 → 차이 체험
2. **데모 3** 직접 실행: Claude Code 자체에 대해 뭐든 질문

> "위 내용을 직접 실행해보세요. 실행이 끝나면 **'완료'** 또는 **'다음'**이라고 입력해주세요."

## QUIZ

```

1. Skill 실행 → 한 마디로 반복 업무 자동화
