# Block 3: 쉬어가기 — 터미널 & 상태 표시줄

> 이 블록은 퀴즈 없이 가볍게 진행한다. Phase A만 있고 Phase B는 없다.
> 사용자가 "완료" 또는 "다음"이라고 하면 Block 3-5 (Agent Teams)로 넘어간다.

## EXPLAIN

여기까지 Claude Code의 핵심 기능 4개(Memory, Skill, MCP, Subagent)를 배웠습니다. 잠깐 숨 고르기.

Claude Code는 **터미널(명령어 창)**에서 돌아갑니다.
어떤 터미널을 쓰느냐에 따라 경험이 달라집니다.

---

### 터미널 추천

| 터미널 | 특징 | 설치 링크 |
|--------|------|----------|
| **Ghostty** | 빠르고 가벼움. GPU 가속. 오픈소스 | [ghostty.org](https://ghostty.org) |
| **iTerm2** | macOS 대표 터미널. 기능 풍부 | [iterm2.com](https://iterm2.com) |
| **Warp** | AI 내장 터미널. 명령어 자동완성 | [warp.dev](https://warp.dev) |

> 💡 **용어 정리**
> - **GPU 가속**: 그래픽 카드를 써서 더 빠르게 화면을 렌더링. 터미널이 더 부드럽고 빠르게 표시됩니다
> - **오픈소스**: 누구나 코드를 볼 수 있는 무료 공개 소프트웨어

기본 Terminal.app도 쓸 수 있지만, 위 터미널들은 색상, 폰트, 화면 분할 등이 훨씬 자유롭습니다.

---

### Claude Code 상태 표시줄 (Status Line)

Claude Code 화면 맨 아래에 **상태 표시줄**을 커스터마이즈할 수 있습니다.
지금 쓰는 모델 이름, 대화 기억 용량 사용량, 현재 브랜치 등을 실시간으로 볼 수 있습니다.

> 💡 **용어 정리**
> - **상태 표시줄(화면 아래 정보 바)**: 현재 상태를 한눈에 보여주는 줄
> - **컨텍스트 사용률(대화 기억 용량 사용 비율)**: 100%에 가까워지면 앞의 대화를 잊기 시작합니다

**가장 쉬운 설정 방법:**

```
/statusline 모델 이름과 컨텍스트 사용률을 보여줘
```

이 한 줄이면 Claude가 알아서 설정 파일을 만들고 적용까지 해줍니다.
