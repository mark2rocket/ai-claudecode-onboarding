## EXPLAIN

### Hook이란?

특정 **사건이 발생하면 자동으로 실행되는 규칙**입니다.

Claude의 응답은 확률적입니다 (매번 조금씩 달라질 수 있음).
반면 Hook으로 설정한 스크립트는 **100% 확실하게** 특정 시점에 실행됩니다.

> 💡 **용어 정리**
> - **Hook(자동 실행 규칙)**: "이 일이 일어나면, 저 명령을 실행해"라고 미리 정해두는 것
> - **스크립트**: 컴퓨터에 내릴 명령들을 파일에 미리 적어둔 것. 요리 레시피처럼 순서대로 실행됩니다
> - **결정론적(100% 확실하게)**: 같은 상황에서 항상 같은 결과가 나오는 것. Claude의 응답(매번 조금씩 다름)과 반대 개념

---

### 비유로 이해하기

> 스마트홈의 "자동화" 기능과 같습니다.
> "문을 열면 → 거실 불 켜기" 처럼
> "Claude 응답 완료 → 현재 시각 출력"

---

### Hook의 종류

| Hook 이름 | 발생 시점 | 예시 활용 |
|-----------|----------|----------|
| `PreToolUse` | Claude가 도구 사용 전 | 위험한 명령 차단 |
| `PostToolUse` | Claude가 도구 사용 후 | 작업 완료 로그 기록 |
| `Stop` | Claude 응답 완료 시 | 완료 시각 표시, 알림음 |
| `SessionStart` | 대화 시작 시 | 날씨/일정 자동 불러오기 |

---

### 라이프스타일 크리에이터 활용 예시

| Hook | 효과 |
|------|------|
| Claude가 캡션 작성 완료 → 소요 시간 자동 기록 | 내 작업 패턴 파악 |
| 새 대화 시작 → 오늘 예정된 업로드 일정 자동 표시 | 매일 일정 놓치지 않기 |
| Claude 응답 완료 → 알림음 재생 | 딴짓하다 돌아올 때 편리 |

---

## EXECUTE

간단한 Stop Hook을 만들어봅니다.
Claude에게 이렇게 요청하세요:

```
Claude가 응답을 완료할 때마다 현재 시각을 터미널 하단에 표시하는 Stop Hook을 만들어줘
```

Claude가 `~/.claude/settings.json`에 Hook 설정을 추가하고 스크립트 파일을 만들어줄 겁니다.

> "위 내용을 직접 실행해보세요. 실행이 끝나면 **'완료'** 또는 **'다음'**이라고 입력해주세요."
