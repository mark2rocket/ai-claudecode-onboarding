### Skill이란?

```
CLAUDE.md ━━━━━━━━━━━━━━━━━━━━━━━━ 항상 로딩 (매 세션)

Skill A   ─ ─ ─ ─ ─ ┐
Skill B   ─ ─ ─ ─ ─ ─ ─ ─ ┐         필요할 때만 로딩
Skill C   ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐
                    ▼         ▼         ▼
               "/sync" 호출  자동 매칭  "/wrap" 호출
```

CLAUDE.md는 항상 전부 읽히지만, Skill은 **호출할 때만** 꺼내 읽힌다.
컨텍스트 윈도우(Claude의 작업 공간)는 유한하니까, 필요한 것만 로딩하는 게 핵심이다.

---

### Skill 구조

```
my-skill/
├── SKILL.md       ← 필수. 스킬의 본체 (지금 이 파일과 같은 구조)
├── references/    ← 선택. 참고 문서
└── scripts/       ← 선택. 실행할 코드
```

> 지금 이 온보딩 자체가 `SKILL.md` + `references/` 구조로 만들어져 있다.
