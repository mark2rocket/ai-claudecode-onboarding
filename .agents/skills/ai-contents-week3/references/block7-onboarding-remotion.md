### 온보딩 영상이란?

```
온보딩 영상 = 새로운 사용자가 제품을 처음 접할 때
               "어떻게 쓰는지"를 빠르게 이해시키는 영상

목표:
  - 핵심 기능을 3분 이내로 전달
  - "아하!" 순간을 최대한 빨리 만들어주기
  - 이탈 없이 첫 성공 경험으로 안내

언제 사용하나:
  - SaaS 제품 가입 후 첫 화면
  - 유튜브/제품 페이지의 "시작하기" 영상
  - 이메일 onboarding 시퀀스
  - 고객 지원 비용 절감
```

---

### 온보딩 영상 구조

```
효과적인 온보딩 5단계:
┌──────────────────────────────────────────┐
│ 0~15초:   웰컴 + 오늘 배울 것 예고      │
│ 15~60초:  Step 1 - 첫 번째 핵심 기능    │
│ 60~120초: Step 2 - 두 번째 핵심 기능    │
│ 120~160초: Step 3 - 세 번째 핵심 기능   │
│ 160~180초: 다음 단계 안내 + CTA         │
└──────────────────────────────────────────┘

각 Step의 패턴:
  1. "이 기능으로 [문제]를 해결할 수 있어요"
  2. 화면 데모 (스크린샷 또는 화면 녹화 삽입)
  3. "이렇게 하면 [결과]가 나와요"
```

---

### 온보딩 영상에서 Claude가 만들어주는 것

```
Claude에게 "온보딩 영상 Remotion 컴포넌트 만들어줘"라고 하면
자동으로 처리해준다:

  - 상단 진행 표시기 (Step 1 → 2 → 3 순서 바)
  - 스크린샷이 왼쪽에서 슬라이드인으로 등장
  - 설명 텍스트가 스크린샷 옆에 부드럽게 나타남
  - UI 요소를 가리키는 화살표 포인터 애니메이션

코드를 이해할 필요 없다.
에러가 나면 에러 메시지를 그대로 Claude에게 붙여넣으면 된다.
```

---

### 화면 캡처 + Remotion 조합

```
온보딩 영상에서 쓸 수 있는 소스:

1. 정적 스크린샷   → 제품 화면 캡처 (Command+Shift+4)
                     Remotion에서 이미지로 삽입해줌

2. 화면 녹화 영상  → QuickTime으로 녹화 → Remotion에 삽입

3. 직접 재현       → 스크린샷 없이도 UI처럼 보이는 효과 제작 가능

Claude 프롬프트 패턴:
  "screenshot.png를 슬라이드인 효과로 보여주는 컴포넌트 만들어줘.
   왼쪽에서 오른쪽으로 나타나게."
```

## EXECUTE

**Step 1 — 온보딩 영상 브리프 작성**

```
onboarding_brief.md를 만들어줘.

---
# 온보딩 영상 브리프

## 제품/서비스
[제품 이름]

## 타깃 사용자
[어떤 사람이 처음 쓰는가?]

## 오늘 배울 핵심 기능 3가지
1. [기능 1] - [이 기능으로 할 수 있는 것]
2. [기능 2] - [이 기능으로 할 수 있는 것]
3. [기능 3] - [이 기능으로 할 수 있는 것]

## 영상 길이
3분 (180초)

## 구조
0~15초:   웰컴 "오늘 [기능 3가지] 배워볼게요"
15~60초:  Step 1 설명 + 화면 데모
60~120초: Step 2 설명 + 화면 데모
120~160초: Step 3 설명 + 화면 데모
160~180초: "이제 직접 해보세요!" + CTA

## 에셋
- screenshot1.png: [기능 1] 화면 캡처
- screenshot2.png: [기능 2] 화면 캡처
- screenshot3.png: [기능 3] 화면 캡처
---
```

---

**Step 2 — 화면 캡처 준비**

```
Mac: Command+Shift+4 → 드래그로 영역 선택 → 저장
각 핵심 기능 화면을 캡처하고 아래 이름으로 저장:
  screenshot1.png
  screenshot2.png
  screenshot3.png

Remotion 프로젝트의 public/ 폴더에 복사:
  cp screenshot*.png my-promo-video/public/
```

---

**Step 3 — 온보딩 영상 컴포넌트 생성**

```
onboarding_brief.md를 읽고
Remotion으로 3분짜리 온보딩 영상 컴포넌트를 만들어줘.

- WelcomeScene: 오늘 배울 것 예고 (상단 진행 표시기 포함)
- StepScene * 3: 각 단계마다 설명 텍스트 + screenshot 슬라이드인
- CTAScene: 다음 단계 안내

요구사항:
- 상단에 Step 1/2/3 진행 표시기
- 각 스크린샷은 왼쪽에서 슬라이드인
- 설명 텍스트는 스크린샷 오른쪽에 fadeIn
- 1920×1080 (16:9)
- src/Onboarding.tsx에 저장
```

---

**Step 4 — 렌더링**

```
"Remotion으로 온보딩 영상을 렌더링해줘.
 출력 파일: out/onboarding-video.mp4"
```

체크리스트:
```
□ onboarding_brief.md가 생성됐나?
□ 스크린샷 3개가 public/ 폴더에 있나?
□ 브라우저 미리보기에서 3단계 구조가 보이나?
□ 상단 진행 표시기가 Step에 따라 변하나?
□ out/onboarding-video.mp4가 렌더링됐나?
```
