### 핵심 개념: HTML 애니메이션 = 소스 없는 영상

```
일반적인 영상 제작:
  카메라 촬영 → 편집 앱 → 영상 파일

코딩으로 만드는 영상:
  Claude → HTML/CSS 애니메이션 → 브라우저 → 화면 녹화 → 영상 파일

장점:
  - 카메라, 편집 앱 불필요
  - 텍스트·색상·타이밍 수정이 코드 한 줄
  - 브랜드 색상/폰트 정확히 적용 가능
  - 완벽하게 재현 가능 (매번 같은 결과)
```

---

### 세로형 영상의 기본 스펙

```
릴스 표준 스펙:
┌──────────────────────────────────────────┐
│  크기:   1080 × 1920px (9:16 비율)       │
│  길이:   15초 ~ 60초 (짧을수록 좋음)     │
│  텍스트: 크고 굵게 (모바일 화면 기준)    │
│  여백:   위아래 15% 안전 영역 확보       │
│  속도:   장면 전환 0.3~0.5초             │
└──────────────────────────────────────────┘
```

---

### CSS 애니메이션 3가지 패턴

```
1. fadeIn (텍스트 등장)
   opacity: 0 → 1
   → 핵심 메시지를 부드럽게 보여줄 때

2. slideUp (아래에서 올라오기)
   transform: translateY(50px) → translateY(0)
   → 리스트 항목을 순서대로 보여줄 때

3. highlight (배경 강조)
   background: transparent → 포인트 컬러
   → 핵심 키워드를 강조할 때
```

### 화면 녹화 방법

```
방법 1 (기본): Playwright 자동 녹화
  Claude에게 입력:
  "reels.html을 Playwright로 열어서 15초 동안
   1080×1920 크기로 녹화하고 reels.mp4로 저장해줘."
  → Claude가 브라우저 실행 + 녹화 + 파일 저장까지 자동 처리
  → 매번 동일한 결과, 수동 조작 불필요

방법 2 (백업): QuickTime Player
  1. QuickTime 열기 → 파일 > 새 화면 녹화
  2. 브라우저에서 HTML 전체화면 (Command+Shift+F)
  3. 녹화 시작 → 애니메이션 완료 → 중지
  4. .mov 저장 후 FFmpeg으로 .mp4 변환

결과 파일: .mp4 → 인스타그램 바로 업로드 가능
```

**Step 3 — Playwright로 영상 녹화**

```
Claude에게 입력:
"reels.html을 Playwright로 열어서 15초 동안
 1080×1920 크기로 녹화하고 reels.mp4로 저장해줘.
 녹화 후 ffmpeg으로 H.264 mp4로 변환까지 해줘."

⚠️ 중요: Playwright는 내부적으로 webm으로 녹화한다.
반드시 ffmpeg으로 변환해야 macOS에서 정상 재생된다:
ffmpeg -i input.webm -vcodec libx264 -acodec aac output.mp4

(Playwright 없을 때 백업:
 브라우저 전체화면 → Command+Shift+5 → 15초 녹화 → 중지)
```

체크리스트:
```

□ 애니메이션이 순서대로 재생되나?
□ 화면 녹화 파일이 저장됐나?
```
