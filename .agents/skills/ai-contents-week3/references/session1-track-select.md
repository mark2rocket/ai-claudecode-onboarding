### Track 1: 릴스 (Instagram/TikTok)

```
목적:  소셜미디어 팔로워 확보, 브랜드 인지도
형식:  세로형 (1080×1920, 9:16)
길이:  15초 ~ 90초
도구:  HTML/CSS/JS + FFmpeg (코딩 중심)
소스:  코딩만 → GIF → 영상 촬영 → 자막/음성 (단계별 추가)

학습 경로:
  Session 2: 코딩만으로 짧은 릴스 (HTML 애니메이션)
  Session 3: GIF + 코딩으로 긴 릴스
  Session 4: GIF + 영상소스로 긴 릴스 (FFmpeg)
  Session 5: 자막 + 음성 완성 릴스 (SRT + TTS)

적합한 사람:
  - 인스타그램/틱톡에서 콘텐츠를 올리는 크리에이터
  - 영상 편집 앱 없이 코딩으로 영상을 만들고 싶은 사람
  - 짧고 임팩트 있는 영상을 빠르게 만들고 싶은 사람
```

### Track 2: Remotion (제품/강의/홍보)

```
목적:  제품 홍보, 사용자 온보딩, 지식 전달
형식:  가로형 (1920×1080, 16:9) 또는 커스텀
길이:  30초 ~ 10분
도구:  Remotion (React 기반 영상 프레임워크)
소스:  React 컴포넌트 + 애니메이션 = 영상

학습 경로:
  Session 6: 홍보 영상 (Hook → 문제 → 해결 → CTA)
  Session 7: 온보딩 영상 (단계별 제품 설명)
  Session 8: 강의 영상 (슬라이드 + 코드 애니메이션)

적합한 사람:
  - 제품/서비스를 만들고 홍보 영상이 필요한 사람
  - 새로운 사용자에게 제품을 소개하는 온보딩 영상
  - 기술/지식을 영상으로 전달하는 강의 크리에이터
```

### 왜 CapCut·Canva가 아닌 HTML과 Remotion인가?

```
CapCut, Canva Video 같은 툴은 훌륭하지만 한 가지 근본적 한계가 있다:
  반드시 사람의 손이 타야 한다.

  클릭 → 드래그 → 텍스트 입력 → 렌더링 버튼 클릭
  → 이 모든 과정을 사람이 매번 직접 해야 한다.

Claude(AI 에이전트)가 처음부터 끝까지 혼자 실행할 수 없다는 뜻이다.

HTML과 Remotion은 다르다:
  Claude가 코드를 작성 → 코드 파일이 곧 영상
  → video_brief.md 하나로 전체 파이프라인 자동 실행 가능
  → Week 4에서 "/my-reels" 한 줄로 전체 과정 자동화

Week 3에서 이 두 가지를 배우는 이유가 여기에 있다.
에이전트 기반 콘텐츠 제작의 핵심은 Claude가 직접 실행할 수 있는 도구다.
```
