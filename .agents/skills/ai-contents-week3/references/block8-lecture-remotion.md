### 강의 영상 vs 온보딩 영상 차이

```
온보딩 영상:
  목적: 제품 사용법을 빠르게 이해
  길이: 3분 이내
  깊이: 얕게, 빠르게
  구조: 단계별 UI 안내

강의 영상:
  목적: 특정 개념이나 기술을 깊이 학습
  길이: 5~20분
  깊이: 깊게, 천천히
  구조: 개념 → 설명 → 예시 → 실습
```

---

### 강의 영상 Remotion 핵심 기법

```
1. 타이핑 애니메이션 (코드 등장 효과)
   const typedText = script.slice(0, Math.floor(frame / 2));
   → 프레임마다 한 글자씩 나타나는 타이핑 효과

2. 코드 하이라이팅
   → 코드 블록에서 특정 줄을 순서대로 강조
   → background: rgba(255,255,0,0.3) 으로 노란 하이라이트

3. 단계별 bullet 등장
   → 각 bullet이 프레임 순서대로 fadeIn
   → interpolate(frame, [0, 15], [0, 1]) 개별 적용

4. 화면 분할 레이아웃
   → 왼쪽: 슬라이드 / 오른쪽: 코드 또는 데모
   → 왼쪽: 강사 영상 / 오른쪽: 화면 공유 스타일
```

### 강의 영상 구조 템플릿

```
효과적인 강의 영상 구조 (5분 기준):
┌──────────────────────────────────────────┐
│ 0~15초:   타이틀 슬라이드                │
│            "오늘 배울 것: [주제]"         │
│ 15~60초:  개념 설명 (왜 중요한가?)       │
│ 60~150초: 핵심 내용 1 (이론 + 예시)     │
│ 150~240초: 핵심 내용 2 (이론 + 예시)    │
│ 240~280초: 핵심 내용 3 (이론 + 예시)   │
│ 280~300초: 요약 + 다음 강의 예고        │
└──────────────────────────────────────────┘

각 핵심 내용 슬라이드 패턴:
  1. 개념 제목 (타이핑 애니메이션)
  2. 설명 3줄 (bullet 순서 등장)
  3. 코드 예시 (타이핑 + 하이라이팅)
  4. 결과 / 정리
```

## EXECUTE

**Step 1 — 강의 기획**

```
lecture_brief.md를 만들어줘.

---
# 강의 영상 브리프

## 강의 주제
[배울 개념 또는 기술]

## 타깃 수강생
[어떤 수준의 사람을 위한 강의인가?]

## 학습 목표
이 강의를 보고 나면 [결과]를 할 수 있다.

## 강의 구성 (5분)
0~15초:   타이틀: "[주제]의 핵심을 5분에"
15~60초:  개념: [왜 이 주제가 중요한가?]
60~150초: 핵심 1: [개념 + 코드 예시]
150~240초: 핵심 2: [개념 + 코드 예시]
240~280초: 핵심 3: [개념 + 코드 예시]
280~300초: 요약 + 다음 강의 예고

## 코드 예시 언어
[Python / JavaScript / Shell 등]

## 스타일
배경: 다크 (#1a1a2e)
포인트: #6B4EFF
코드 폰트: Fira Code, JetBrains Mono
---
```

**Step 2 — 강의 영상 컴포넌트 생성**

```
lecture_brief.md를 읽고
Remotion으로 5분짜리 강의 영상 컴포넌트를 만들어줘.

컴포넌트 구조:
- TitleSlide: 타이핑 애니메이션으로 제목 등장
- ConceptSlide: bullet 3개가 순서대로 fadeIn
- CodeSlide * 3: 코드가 한 줄씩 타이핑 + 핵심 줄 노란 하이라이팅
- SummarySlide: 전체 내용 요약 bullet

추가 요구사항:
- 슬라이드 번호 표시 (우하단)
- 각 섹션 사이 0.5초 검정 페이드 전환
- 코드 블록: 어두운 박스, 녹색 코드 텍스트
- src/Lecture.tsx에 저장
```
