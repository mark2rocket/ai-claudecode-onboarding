---

### HTML이 크리에이터의 마스터 포맷인 이유

```
┌─ HTML 하나 ──────────────────────────────────┐
│                                              │
│  → 카드뉴스  브라우저 캡처 → PNG → 인스타    │
│                                              │
│  → PDF       브라우저 인쇄 → PDF → 문서      │
│                                              │
│  → PPT       HTML 슬라이드 → 발표 자료       │
│                                              │
│  → 영상      GIF / MP4 변환 → 릴스·유튜브   │
│                                              │
└──────────────────────────────────────────────┘
```

**OSMU (One Source Multi Use):**
하나의 소스(HTML)를 한 번 잘 만들어두면, 다양한 채널에 맞게 변환해서 쓸 수 있다.

---

### Canva·파워포인트 대신 HTML을 쓰는 이유

| 도구 | 장점 | 단점 |
|------|------|------|
| Canva | 쉽고 예쁨 | AI가 직접 편집 불가. 매번 수동 작업 |
| 파워포인트 | 익숙함 | 자동화 어려움. 포맷 변환 번거로움 |
| **HTML** | **Claude가 직접 생성·수정·변환** | 처음엔 낯설 수 있음 |

> Claude가 만들고, 수정하고, 다른 포맷으로 변환까지 해준다.
> 사람이 할 일은 "이렇게 바꿔줘" 한 마디뿐.

## EXECUTE

맛보기로 Claude에게 직접 물어보자:

```
카드뉴스 1장짜리 HTML을 만들어줘.
제목은 "AI 시대 크리에이터의 무기", 배경색 #6B4EFF, 흰 텍스트.
브라우저로 바로 열어줘.
```

> HTML이 브라우저에 열리면 — 이게 카드뉴스의 시작이다.
> Session 0~7에서 이 HTML을 점점 정교하게 만들어간다.
