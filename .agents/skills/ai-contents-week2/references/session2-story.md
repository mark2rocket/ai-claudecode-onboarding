### 카드뉴스 스토리 구조

```
┌─────────────────────────────────────────┐
│  카드뉴스 기본 스토리 구조 (5장 기준)   │
│                                         │
│  1장: 문제제기   "왜 이걸 봐야 하나?"  │
│  2장: 본론 1     핵심 내용 첫 번째     │
│  3장: 본론 2     핵심 내용 두 번째     │
│  4장: 인사이트   "그래서 어떻게?"      │
│  5장: 행동(CTA)  "지금 당장 이것만"   │
└─────────────────────────────────────────┘
```

이 슬롯에 내 주제를 배정하면 Claude가 내용을 채운다.

### 후킹 — 클릭하게, 끝까지 읽게

**제목 후킹:**
```
밋밋한 제목:  "AI 트렌드 정리"
후킹 제목:   "내 직장이 AI로 바뀐다 — 30대가 지금 알아야 할 것"
             "AI 모르면 3년 후 뒤처진다 (이미 시작됐다)"
```

**첫 카드 후킹 (문제제기):**
```
밋밋한 시작:  "AI란 무엇인가요?"
후킹 시작:   "요즘 회의에서 AI 얘기 나올 때 혼자 조용한 적 있나요?"
             → 독자가 "나 얘기네" 하고 끝까지 읽는다
```

**후킹이 통하는 이유:**
```
정보 나열  →  "나중에 읽어야지" → 저장만 하고 안 읽음
공감/문제  →  "나 얘기잖아"    → 끝까지 읽고 저장하고 공유
```

### 콘텐츠 인풋 2가지 트랙

스토리 구조가 정해지면 내용을 어디서 가져오느냐에 따라 두 가지 트랙:

```
Track A (리서치): Claude가 최신 트렌드·데이터·사례를 직접 수집
→ "최신 트렌드와 실제 사례를 리서치해서 채워줘"

Track B (인터뷰): Claude가 나에게 질문해서 내 경험으로 내용 구성
→ "내 경험을 인터뷰해서 내용을 뽑아줘"
```

## EXECUTE

**트랙 선택 후 실행:**

**Track A (리서치):**
```
30대 직장인을 위한 AI 트렌드 카드뉴스 5장 만들어줘.
1장: 문제제기 (공감 유도 후킹으로 시작),
2장: AI가 바꾸는 직장 생활,
3장: 당장 써볼 수 있는 AI 도구 3가지,
4장: AI 시대 직장인 생존 전략,
5장: 마무리+CTA.
제목은 클릭하고 싶게 후킹으로 만들어줘.
최신 트렌드와 사례를 리서치해서 채워줘.
```

**Track B (인터뷰):**
```
30대 직장인을 위한 AI 활용 경험 카드뉴스 5장을 만들려고 해.
1장: 문제제기, 2~4장: 내 실제 경험, 5장: CTA.
제목 후킹도 같이 만들어줘.
내용을 채우기 위해 나에게 질문해줘.
```

> Session 1 결과와 비교: 스토리 흐름이 생겼는지, 제목이 더 당기는지 확인.
