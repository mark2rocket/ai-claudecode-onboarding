---

### 디자인 스펙 4요소

```
┌───────────────────────────────────────────────────┐
│  요소      값            이유                     │
├───────────────────────────────────────────────────┤
│  크기      1080×1080px   인스타그램 정사각형 표준  │
│  색상      브랜드 컬러   일관된 시각 아이덴티티    │
│  폰트      고딕 계열     모바일 가독성 최적         │
│  헤드라인  15자 이내     모바일에서 한눈에 읽힘    │
└───────────────────────────────────────────────────┘
```

브랜드 컬러가 없으면 기본값:
```
배경색: #6B4EFF  (보라 계열)
텍스트: #FFFFFF  (흰색)
폰트:   Noto Sans KR
```

---

### 레이아웃 — 카드 유형별 배치

```
┌──────────────────────────────────────────────────┐
│  유형        레이아웃 특징                       │
├──────────────┬───────────────────────────────────┤
│  1장 (표지)  │ 제목 중앙 정렬, 상하 여백 넉넉히  │
│              │ 부제는 제목 크기의 50%            │
├──────────────┼───────────────────────────────────┤
│  2~4장 (본문)│ 좌측 상단 넘버링(01/02/03)        │
│              │ 제목 상단 고정                    │
│              │ 핵심 요약 하단 고정               │
│              │ 좌우 패딩 80px                    │
├──────────────┼───────────────────────────────────┤
│  5장 (CTA)   │ 전체 중앙 정렬                    │
│              │ CTA 문구 버튼 스타일              │
│              │ 여백 최대화                       │
└──────────────┴───────────────────────────────────┘
```

카드마다 역할이 다르니 레이아웃도 다르다.
표지는 임팩트, 본문은 가독성, CTA는 행동 유도.

## EXECUTE

**내 스펙 결정 후 실행:**

```
배경색: _______  (브랜드 컬러 또는 #6B4EFF)
폰트:   _______  (Noto Sans KR 또는 Pretendard)
```

**프롬프트:**

```
30대 직장인을 위한 AI 트렌드 카드뉴스 5장 스펙을 정의해줘.

디자인 스펙:
- 크기: 1080×1080px
- 배경색: #6B4EFF, 텍스트: #FFFFFF
- 폰트: Noto Sans KR, 헤드라인 15자 이내

레이아웃:
- 1장(표지): 전체 중앙 정렬, 상하 여백 120px, 부제 폰트 크기 50%
- 2~4장(본문): 좌측 상단 넘버링(01/02/03), 제목 상단 고정,
               핵심 요약 하단 고정, 좌우 패딩 80px
- 5장(CTA): 전체 중앙 정렬, CTA 문구 버튼 스타일, 여백 최대화

이 스펙을 content_brief.md에 저장해줘.
```

> content_brief.md에 저장해두면 Session 4(HTML)에서 바로 불러다 쓴다.
