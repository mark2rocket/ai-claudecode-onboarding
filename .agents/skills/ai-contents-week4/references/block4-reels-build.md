### 릴스 Human-in-the-loop 체크 포인트

릴스는 카드뉴스보다 확인해야 할 것이 더 많다. HTML 단계에서 아래 3가지를 반드시 점검한다.

```
Phase 1 (자동 실행):
  브리프 입력
  → brand.md + style.md 로드 (자동)
  → HTML 릴스 생성
  → 브라우저 열기
  → ⛔ STOP: 아래 체크리스트 출력 후 기다린다.

  "아래 항목을 확인해주세요:
   ✓ 텍스트 크기가 모바일에서 읽히는가 (80px 이상)
   ✓ 애니메이션 타이밍이 자연스러운가 (너무 빠르거나 느리지 않게)
   ✓ 색상이 brand.md와 맞는가

   수정할 부분을 말씀해주세요. 없으면 '확인완료'라고 입력해주세요."

Phase 2 (확인완료 입력 후):
  수정사항 반영
  → 화면녹화 방법 안내
  → MP4 완성
```

---

### 화면녹화 체크리스트 (녹화 전 필수)

| 항목 | 방법 |
|------|------|
| 녹화 도구 | macOS: QuickTime Player 또는 Cmd+Shift+5 |
| 브라우저 창 크기 | 1080×1920에 최대한 맞추거나 브라우저 줌 조절 |
| 녹화 전 준비 | HTML 새로고침 (Cmd+R) 후 처음 장면에서 시작 |
| 오디오 | 배경음악 추가 시 QuickTime 오디오 설정 확인 |

> 화면 크기를 정확히 맞추기 어렵다면 녹화 후 FFmpeg으로 크롭/리사이즈할 수 있다.

### SKILL.md 완성 예시 (릴스)

```markdown
---
name: 내릴스
description: 내 브랜드 릴스 자동 생성. "/내릴스 주제" 형태로 호출.
---

# 내릴스 실행 순서

## Phase 1 (자동 실행)
1. references/brand.md 읽기
2. references/style.md 읽기
3. references/brief.md 프롬프트로 릴스 브리프 작성
4. references/style.md 스펙으로 HTML 릴스 생성
5. 브라우저로 HTML 열기
6. ⛔ STOP: 체크리스트 출력 후 기다린다.

## Phase 2 (확인완료 입력 후)
7. 수정사항 반영
8. 화면녹화 방법 안내:
   - Cmd+Shift+5 → 화면 영역 선택 → 브라우저 창
   - HTML 새로고침 후 녹화 시작
9. "MP4 파일을 저장했다면 완성입니다."
```

## EXECUTE

**1. references/brief.md 작성 (릴스 전용)**

```bash
cat > ~/.claude/skills/내릴스/references/brief.md << 'EOF'
# 릴스 브리프 프롬프트

아래 정보로 인스타그램 릴스 브리프를 작성해줘.

주제: [사용자 입력값]
brand.md와 style.md의 스펙을 반드시 반영할 것.

브리프 형식:
- 핵심 메시지: (한 줄로)
- 장면 수: 5~8장면
- 각 장면: 핵심 텍스트 + 애니메이션 방향
- 첫 장면: 훅 — 스크롤을 멈추게 하는 강한 한 줄
- 마지막 장면: CTA (팔로우 / 저장 / 링크)
EOF
```

**2. references/workflow.md 작성**

```bash
cat > ~/.claude/skills/내릴스/references/workflow.md << 'EOF'
# 릴스 워크플로우

## HTML 생성 규칙
- 크기: style.md 기준 (1080×1920)
- 각 장면을 별도 div로 구분, CSS animation으로 전환
- 텍스트는 반드시 80px 이상

## 브라우저 확인 후 STOP
HTML 생성 후 브라우저에서 열고 아래 문구 출력:
"텍스트 크기 / 애니메이션 타이밍 / 색상을 확인해주세요.
수정할 부분 말씀해주세요. 없으면 '확인완료' 입력해주세요."

## 확인완료 후
화면녹화 방법 안내:
- Cmd+Shift+5 → 브라우저 창 선택
- HTML 새로고침 후 녹화 시작
- 녹화 완료 후 MP4 저장

## 절대 하지 말 것
- 확인 없이 바로 녹화 안내 금지
- 40px 미만 텍스트 생성 금지
- 가로형(16:9) HTML 생성 금지 (반드시 9:16 세로)
EOF
```

```bash
cat > ~/.claude/skills/내릴스/SKILL.md << 'EOF'
---
name: 내릴스
description: 내 브랜드 릴스 자동 생성. "/내릴스 주제" 형태로 호출.
---

# 내릴스 실행 순서

## Phase 1 (자동 실행)
1. references/brand.md 읽기
2. references/style.md 읽기
3. references/brief.md 프롬프트로 릴스 브리프 작성
4. style.md 스펙 + brief 내용으로 HTML 릴스 생성
5. 브라우저로 HTML 열기
6. ⛔ STOP: 아래 체크리스트 출력 후 기다린다.
   "아래 항목을 확인해주세요:
    ✓ 텍스트 크기가 모바일에서 읽히는가 (80px 이상)
    ✓ 애니메이션 타이밍이 자연스러운가
    ✓ 색상이 브랜드와 맞는가
    수정할 부분을 말씀해주세요. 없으면 '확인완료' 입력해주세요."

## Phase 2 (확인완료 입력 후)
7. 수정사항 반영
8. 화면녹화 방법 안내 (Cmd+Shift+5, HTML 새로고침 후 시작)
9. "MP4 저장 완료 시 릴스 완성입니다."

```


**6. 두 번째 주제로 재실행 → 첫 번째보다 빠른 것 체감**

**체크리스트**
- [ ] brief.md, workflow.md 완성
- [ ] SKILL.md 작성 완료 (Phase 1 STOP 구조 포함)
- [ ] 첫 실행 — HTML 브라우저에서 확인
- [ ] 체크리스트 3항목 점검 (텍스트 크기, 타이밍, 색상)
- [ ] 화면녹화 완료 → MP4 저장
- [ ] 2회째 실행 — 더 빠른 것 체감

## QUIZ

**퀴즈: 릴스 HTML 프리뷰에서 반드시 확인해야 할 가장 중요한 항목은?**

1. HTML 파일 용량 (10MB 초과 시 브라우저가 느려짐)
2. 모바일에서 텍스트가 읽히는 크기인가 (40px 미만이면 실제 화면에서 안 보임)
3. HTML 문법 오류 여부 (W3C 유효성 검사)

정답: 2번. 릴스는 스마트폰 세로 화면에서 소비된다. 아무리 내용이 좋아도 텍스트가 너무 작아서 안 읽히면 스크롤을 내려버린다. 80px 이상이 기준이고, 40px 미만은 실제 릴스에서 읽기 어렵다. HTML 단계에서 반드시 확인해야 녹화 후 재작업을 막을 수 있다.
