### 최적화 팁 1: 금지 목록 추가

"원하는 것"보다 "원하지 않는 것"이 Claude를 더 빠르게 교정한다.

references/pipeline.md (또는 workflow.md)의 금지 목록에 추가한다:

```markdown
## 절대 하지 말 것
- 영어 단어 섞지 말 것 (예: "마케팅 insight" → "마케팅 인사이트"도 금지, "마케팅 핵심"으로)
- 이모지 3개 초과 금지
- 카드 7장 초과 금지
- "~습니다" 체 금지 (구어체로: "~다", "~해요")
- "혁신적인", "효과적으로", "이를 통해" 사용 금지
```

금지 목록은 많을수록 좋다. 결과물에서 마음에 안 들었던 것들을 그때그때 추가한다.

---

### 최적화 팁 2: 특정 부분만 재생성

전체 재실행 없이 맘에 안 드는 부분만 수정한다:

```
"3번 카드 헤드라인만 다시 써줘. 15자 이내로. 나머지는 그대로."
"마지막 카드 CTA 문구를 '팔로우'에서 '저장'으로 바꿔줘."
"2번 장면 애니메이션만 더 빠르게 조정해줘."
```

전체를 다시 만드는 것보다 훨씬 빠르고, 나머지 부분이 망가지지 않는다.

---

### 최적화 팁 3: CSS 변수 수정

HTML 파일 상단의 `:root` 블록에서 변수만 바꾸면 전체 디자인이 일괄 변경된다.

```css
/* HTML 파일 맨 위 <style> 태그 안 */
:root {
  --main-color: #1A50E0;    /* 여기만 바꾸면 전체 메인 색상 변경 */
  --sub-color: #F5F5F5;
  --font-size-h1: 96px;     /* 제목 폰트 크기 일괄 조정 */
  --font-size-body: 48px;
  --card-padding: 60px;     /* 여백 일괄 조정 */
}
```

매번 Claude에게 "색상 바꿔줘"를 요청하지 않고, 파일을 직접 열어서 `:root` 블록만 수정하면 된다.

### 최적화 팁 4: CLAUDE.md 연동

프로젝트 루트(또는 홈 디렉토리)의 CLAUDE.md에 브랜드 기본값을 추가하면, 스킬의 brand.md와 이중으로 작동한다.

```markdown
# CLAUDE.md에 추가

## 콘텐츠 제작 기본값
- 브랜드명: [내 브랜드]
- 계정: @[계정명]
- 톤: 짧고 직설적, 구어체
- 금지어: "혁신적인", "효과적으로", "이를 통해", "시너지"
- 이모지: 최대 3개
```

CLAUDE.md는 어떤 스킬을 실행하든 항상 읽힌다. brand.md보다 더 넓은 범위에서 일관성을 유지할 수 있다.

### 최적화 팁 5: Plan mode 선활용

실행 전 Claude의 계획을 먼저 확인한다.

```
Shift+Tab → Plan mode 켜기 → /my-cardnews 주제 입력
```

Claude가 "이렇게 실행하겠습니다"라는 계획을 먼저 보여준다. 방향이 다르면 실행 전에 수정할 수 있다.

잘못된 방향으로 끝까지 실행한 다음 "다시 해줘"를 반복하는 낭비를 없앤다.

### 최적화 팁 6: JSONL로 원인 추적 (심화)

결과가 기대와 달랐을 때 Claude가 어떻게 해석했는지 역추적할 수 있다.

```bash
# Claude Code 세션 파일 확인
ls ~/.claude/projects/[프로젝트명]/
# .jsonl 파일 확인

# Claude 응답 내용 일부 확인
cat [세션파일].jsonl | grep "assistant" | head -20
```

어느 단계에서 해석이 달라졌는지 정확히 찾으면, 그 단계의 프롬프트(references 파일)만 수정하면 된다. 전체를 다시 짜지 않아도 된다.

### 최적화 팁 7: 템플릿 버전 관리

같은 스킬을 여러 스타일 버전으로 관리하고 싶을 때 사용한다.

**Step 1 — 신규 템플릿 이름으로 저장:**
```
/my-cardnews template-01 이름으로 신규 템플릿 만들거야
```

Claude가 현재 design.md 기반으로 `references/templates/template-01/design.md`를 생성한다.
이후 같은 스타일이 필요하면 항상 이 템플릿을 호출하면 된다.

**Step 2 — 저장된 템플릿으로 실행:**
```
/my-cardnews template-01 템플릿으로 "Openclaw" 주제로 만들어줘
```

SKILL.md가 `template-01` 키워드를 감지해서 `references/templates/template-01/design.md`를 로드하고,
그 스펙으로 HTML을 생성한다.

| 템플릿 | 용도 예시 |
|--------|----------|
| template-01 | 기본 브랜드 스타일 (현재 design.md 기준) |
| template-02 | 레퍼런스 이미지 기반 스타일 (팁 8에서 생성) |
| template-03 | AI티 없는 심플 스타일 (팁 9에서 생성) |

> templates/ 폴더 안에 각각의 design.md를 두면 된다. SKILL.md에서 템플릿 이름을 받아서 해당 design.md를 로드하도록 분기한다.

### 최적화 팁 8: 레퍼런스 이미지로 스타일 복제

마음에 드는 카드뉴스 스크린샷을 찍어서 그 스타일 그대로 만들 수 있다.

**사용법:**
1. 레퍼런스 이미지를 캡처해서 Claude Code 대화에 첨부
2. 아래 프롬프트를 그대로 입력:

```
/my-cardnews 지금까지 만들었던 카드뉴스 디자인과 레이아웃 규칙을 모두 무시하고
첨부한 이미지 그대로 만들어줘 ('simple-01'이라는 이름으로 신규 템플릿 생성해)
```

Claude가 첨부 이미지를 분석해서:
- 배경색, 텍스트 크기, 폰트 굵기 추출
- 레이아웃 구조 (텍스트 위치, 여백) 복제
- 해당 스타일로 HTML 생성
- `references/templates/simple-01/design.md`로 자동 저장

이후 같은 스타일로 다시 만들고 싶으면:
```
/my-cardnews simple-01 템플릿으로 "AI 마케팅 트렌드" 주제로 만들어줘
```

### 최적화 팁 9: AI티 없는 심플 스타일

복잡한 디자인 없이 텍스트 + 이미지만으로 깔끔하게 만들고 싶을 때.

**프롬프트 그대로 복붙해서 쓰면 된다:**
```
/my-cardnews 지금까지 만들었던 카드뉴스 디자인과 레이아웃 규칙을 모두 무시하고
하얀/검은 배경에 헤드 텍스트 1문장, 그리고 바로 아래 서브 텍스트 1-3문장,
그리고 바로 아래에 웹에서 다운받은 이미지 넣어서 만들어줘
('simple-02'이라는 이름으로 신규 템플릿 생성해)
```

Claude가 아래 구조로 HTML을 생성하고 `references/templates/simple-02/design.md`로 저장한다.

**결과물 구조:**
```
┌──────────────────────────┐
│                          │
│   헤드 텍스트 1문장       │ ← 크고 굵게, 화면 상단
│                          │
│   서브 텍스트 1~3문장     │ ← 작고 얇게, 헤드 바로 아래
│                          │
│   ┌──────────────────┐   │
│   │   이미지          │   │ ← 서브 텍스트 바로 아래
│   └──────────────────┘   │
│                          │
└──────────────────────────┘
```

이후 같은 스타일로 반복하고 싶으면:
```
/my-cardnews simple-02 템플릿으로 "1인기업 마케팅 팁" 주제로 만들어줘
```

> AI가 생성한 것처럼 보이지 않는 게 목표다. 텍스트는 짧게, 이미지는 크게. 장식 요소·이모지 없음.
