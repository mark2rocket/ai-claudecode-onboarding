### Week 2에서 손으로 하던 것을 파일로 담는다

Week 2에서 카드뉴스를 만들 때 매번 직접 입력했던 것들이 있다:

- "내 브랜드는 XX이고, 메인 컬러는 #1A50E0이고, 톤은 구어체로..."
- "카드는 7장으로, 첫 카드는 훅 질문으로..."

이것들을 **references/ 파일에 한 번만 저장**해두면, 스킬 실행 시마다 Claude가 자동으로 읽어서 적용한다.

---

### 카드뉴스 스킬 폴더 구조

```
my-cardnews/
├── SKILL.md          ← 실행 순서 지시 (오케스트레이터)
└── references/
    ├── brand.md      ← 내 브랜드 정보 (한 번만 설정)
    ├── brief.md      ← 브리프 작성 프롬프트
    ├── script.md     ← 스크립트 생성 프롬프트
    ├── design.md     ← 디자인 스펙 (색상, 폰트, 레이아웃)
    └── pipeline.md   ← 전체 실행 순서 + 금지 목록
```

---

### brand.md의 역할 — 이 파일이 핵심이다

brand.md는 **내 브랜드 정체성을 한 곳에 저장**하는 파일이다.

나중에 릴스 스킬을 만들 때도 같은 brand.md를 복사해서 쓸 수 있다. 브랜드 정보가 바뀌면 이 파일 하나만 수정하면 두 스킬에 모두 반영된다.

## EXECUTE

**AskUserQuestion으로 브랜드 정보를 수집한 후 brand.md를 자동 생성한다.**

      "options": [

    {

### Step 2 — 폴더 생성 + brand.md 작성

```bash
mkdir -p ~/.claude/skills/my-cardnews/references
```

수집한 답변으로 Write 툴을 사용해 `~/.claude/skills/my-cardnews/references/brand.md`를 생성한다:

```markdown
# 브랜드 정보

브랜드명: [답변1 브랜드명]
인스타 계정: @[답변1 계정명]
메인 컬러: [답변2 색상]
서브 컬러: #F5F5F5
폰트 톤: [답변3 톤]
주요 주제: [Other로 추가 입력한 경우 반영, 없으면 비워둠]
타겟 독자: [Other로 추가 입력한 경우 반영, 없으면 비워둠]
금지 사항: 영어 혼용 금지, 이모지 과다 사용 금지
```

### Step 3 — 생성 확인

파일 내용을 출력해서 보여준다. 수정할 부분이 있으면 말해달라고 안내한다.

**체크리스트**

## QUIZ

**퀴즈: brand.md를 별도 파일로 분리해서 저장하는 가장 큰 이유는?**

1. 파일 크기를 줄이고 Claude의 로딩 속도를 빠르게 하기 위해
2. 카드뉴스·릴스 두 스킬이 같은 브랜드 정보를 공유하고, 한 곳만 수정하면 전체에 반영되기 때문
3. Claude Code 규칙상 브랜드 정보는 반드시 별도 파일에 저장해야 하기 때문

정답: 2번. 브랜드 정보를 한 파일에 모으면 "싱글 소스 오브 트루스(Single Source of Truth)"가 된다. 나중에 계정명이나 컬러가 바뀌어도 brand.md 하나만 고치면 카드뉴스·릴스 스킬 양쪽에 즉시 반영된다.
