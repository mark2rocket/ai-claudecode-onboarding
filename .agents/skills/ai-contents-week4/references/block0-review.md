### 스킬 = 내 콘텐츠 공장

Week 1~3에서 손으로 반복하던 과정들이 있다.

- Week 2: 브리프 작성 → 스크립트 생성 → HTML 만들기 → PNG 렌더링
- Week 3: HTML 애니메이션 → 브라우저 확인 → 화면녹화 → MP4 완성

이 과정을 **스킬 파일에 저장해두면**, 다음부터는 `/my-cardnews 주제` 한 줄로 동일한 파이프라인이 자동 실행된다.

스킬은 **콘텐츠 공장**이다. 한 번 설계하면 계속 쓴다.

---

### 스킬 파일 구조 (기존 week2 스킬 기준)

```
~/.claude/skills/ai-contents-week2/
├── SKILL.md              ← 오케스트레이터 (어떤 순서로 뭘 할지 지시)
└── references/
    ├── block0-review.md  ← 실행 레시피 1
    ├── block1-brief.md   ← 실행 레시피 2
    └── ...
```

**SKILL.md의 역할**: Claude에게 "이 순서로 이 파일들을 읽어서 실행해"라고 지시하는 오케스트레이터.

**references/ 파일들의 역할**: 각 단계에서 실제로 수행할 프롬프트, 스펙, 금지 목록 등을 담은 실행 레시피.

### Week 4에서 만들 것

| 스킬 | 명령어 | 결과물 |
|------|--------|--------|
| my-cardnews | `/my-cardnews 마케팅 팁` | 브랜드 맞춤 카드뉴스 PNG |

## QUIZ

**퀴즈: 스킬에서 SKILL.md의 역할은 무엇인가?**

1. 실제 프롬프트 내용과 브랜드 정보를 직접 저장하는 곳
2. 오케스트레이터 — 어떤 파일을 어떤 순서로 실행할지 Claude에게 지시하는 곳
3. 완성된 콘텐츠 결과물을 저장하는 곳

정답: 2번. SKILL.md는 "지시자" 역할이다. 실제 프롬프트나 브랜드 정보는 references/ 파일들에 나눠 담긴다. SKILL.md는 그 파일들을 어떤 순서로 어떻게 실행할지 Claude에게 알려주는 오케스트레이터다.
