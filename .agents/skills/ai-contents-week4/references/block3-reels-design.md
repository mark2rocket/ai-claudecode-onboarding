### Week 3 Track 1 워크플로우 복습

Week 3에서 릴스를 만들 때 거쳤던 단계:

```
HTML 애니메이션 제작
→ 브라우저에서 프리뷰 확인
→ 화면녹화 (QuickTime / Cmd+Shift+5)
→ (선택) GIF 배경 / 자막 / 음성 합성
→ MP4 완성
```

이 흐름을 **릴스 스킬에 그대로 담는다.**

---

### 릴스 스킬 폴더 구조

```
내릴스/
├── SKILL.md          ← 실행 순서 지시
└── references/
    ├── brand.md      ← 카드뉴스 스킬과 동일 (복사해서 사용)
    ├── brief.md      ← 릴스 브리프 프롬프트
    ├── style.md      ← 릴스 스타일 스펙 (크기, 색상, 애니메이션)
    └── workflow.md   ← HTML → 녹화 단계 안내 + 금지 목록
```

---

### style.md — 릴스 전용 스펙 파일

카드뉴스 스킬의 design.md와 유사하지만, 릴스는 세로형(9:16)이고 애니메이션이 핵심이다.

**반드시 포함해야 할 항목:**

| 항목 | 설명 |
|------|------|
| 크기 | 1080×1920 (9:16 세로) |
| 배경 | 단색 / GIF 사용 여부 |
| 메인 텍스트 크기 | 80px 이상 (모바일에서 읽혀야 함) |
| 애니메이션 | 텍스트 등장 방식, 속도 |
| 자막 위치 | 화면 하단 20% 권장 |
| 금지 사항 | 작은 텍스트(40px 미만), 복잡한 배경 |

---

### brand.md 공유 전략

카드뉴스 스킬에서 이미 만든 brand.md를 그대로 활용한다.

두 가지 방법:

**방법 A (복사)**: brand.md를 복사해서 내릴스/references/에 저장
- 장점: 각 스킬이 독립적
- 단점: 하나 수정 시 다른 것도 수동 업데이트 필요

**방법 B (절대경로 참조)**: SKILL.md에서 `~/.claude/skills/내카드뉴스/references/brand.md` 직접 참조
- 장점: 한 번만 관리
- 단점: 경로가 바뀌면 두 스킬 모두 수정 필요

**Week 4에서는 방법 A(복사)를 쓴다.** 단순하고 안전하다.

## EXECUTE

**1. 릴스 스킬 폴더 생성**

```bash
mkdir -p ~/.claude/skills/내릴스/references
```

**2. brand.md 복사 (카드뉴스 스킬에서)**

```bash
cp ~/.claude/skills/내카드뉴스/references/brand.md \
   ~/.claude/skills/내릴스/references/brand.md
```

**3. references/style.md 작성**

아래 템플릿을 복사해서 내 스타일로 교체한다.

```bash
cat > ~/.claude/skills/내릴스/references/style.md << 'EOF'
# 릴스 스타일 스펙

크기: 1080×1920 (9:16 세로)
배경: #[색상코드] 또는 GIF 사용 (brand.md 메인 컬러 기본)
메인 텍스트 크기: 80px 이상
서브 텍스트 크기: 48px 이상
애니메이션: 텍스트 fade-in, 장면 전환 2초 간격
자막 위치: 화면 하단 20%
자막 색상: 흰색 (#FFFFFF), 검은 테두리 2px
장면 수: 5~8장면
총 재생 시간: 15~30초

금지 사항:
- 40px 미만 텍스트 (모바일에서 안 보임)
- 복잡한 배경 (텍스트 가독성 저하)
- 5초 이상 아무 움직임 없는 장면
EOF
```

**4. 폴더 구조 확인**

```bash
ls -la ~/.claude/skills/내릴스/references/
```

**체크리스트**
- [ ] `내릴스/references/` 폴더 생성
- [ ] brand.md 복사 완료 (내카드뉴스에서)
- [ ] style.md 작성 — 메인 텍스트 80px 이상으로 설정
- [ ] 배경 색상 결정 (단색 or GIF)
- [ ] 금지 사항 최소 2개 입력

## QUIZ

**퀴즈: 릴스 스킬에서 brand.md를 카드뉴스 스킬과 공유하는 가장 현실적인 방법은?**

1. 불가능하다. 각 스킬에 독립적으로 별도 작성해야 한다
2. brand.md를 복사해두면 된다 — 단, 하나를 수정하면 다른 것도 수동으로 업데이트해야 한다
3. 두 스킬이 동일한 brand.md 절대경로를 참조하도록 SKILL.md에 지정한다

정답: 2번 (3번도 정답으로 인정). 2번은 단순하고 안전하다. 파일이 독립적이라 한 스킬에서 실수해도 다른 스킬에 영향을 주지 않는다. 3번은 더 효율적이지만 경로 관리가 필요하다. 두 방법 모두 실제로 쓰인다.
