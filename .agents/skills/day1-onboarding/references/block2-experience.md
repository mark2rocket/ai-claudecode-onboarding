### Before / After

| Before (지금) | After (7일 후) |
|---------------|----------------|
| Slack·Calendar·Notion 하나씩 열어서 30분 | 한 문장으로 자동 발송 |
| 모호하게 요청 → Claude가 추측 | Claude가 질문으로 먼저 명확화 |
| 모르면 문서 뒤지기 | 모르면 Claude에게 물어보기 |

---

### 데모 1: Skill 실행 — `/weekly-sync`

> 강사가 직접 시연한다. 참가자는 보기만 한다.

강사가 `/weekly-sync` 한 줄로 Slack 수집 → git log → 문서 생성을 자동화하는 과정을 보여준다.

**핵심:** 한 마디면 끝. 7일 후에는 여러분도 이런 걸 직접 만든다.

---

### 데모 2: 모호한 요청 → Claude가 질문으로 명확화

먼저 **모호하게** 요청:
```
우리 팀 업무를 개선해줘
```

그다음 **맥락을 추가**해서:
```
우리 팀 업무를 개선해줘. 모호한 부분은 AskUserQuestion으로 질문해서 명확하게 만들어
```

Claude가 "어떤 팀인가요?", "몇 명인가요?" 등 선택지를 제시하며 질문하는 것을 체험한다.

---

### 데모 3: 모르면 Claude에게 물어보기

```
Claude Code에서 MCP가 뭐야?
```

**핵심:** 기능을 다 외울 필요 없다. 모르면 Claude에게 물어보면 된다.
